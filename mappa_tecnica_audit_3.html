CRITICALCaptcha solver: fetchWithRetryPolicy senza rate limiter dedicatosrc/captcha/solver.ts
Root cause: Il VisionSolver usa fetchWithRetryPolicy con maxAttempts:2 e circuitKey:"vision.api". Il circuit breaker si apre dopo N failure consecutive (default 5), ma tra i tentativi NON c'e rate limiting sulla frequenza delle chiamate. Se il captcha appare in loop (es. LinkedIn mostra captcha dopo captcha), il solver viene chiamato ripetutamente dal challengeHandler senza cooldown tra invocazioni.

Impatto: Burn di budget API (Ollama locale = CPU, OpenAI = costi), rallentamento di tutto il sistema, possibile ban IP dall'API provider.
Fix proposto
// In src/captcha/solver.ts - aggiungere rate limiter const CAPTCHA_COOLDOWN_MS = 10_000; // 10s tra tentativi const MAX_CAPTCHA_PER_SESSION = 3; let lastCaptchaAttempt = 0; let captchaAttemptsThisSession = 0; public async analyzeImage(base64Image: string, prompt: string): Promise<string> { const now = Date.now(); if (now - lastCaptchaAttempt < CAPTCHA_COOLDOWN_MS) { throw new Error('Captcha rate limited: cooldown attivo'); } if (captchaAttemptsThisSession >= MAX_CAPTCHA_PER_SESSION) { throw new Error('Captcha max attempts per session raggiunto'); } lastCaptchaAttempt = now; captchaAttemptsThisSession++; // ... rest of implementation }
