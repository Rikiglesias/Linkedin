🤖 Bot LinkedIn — Cheat-sheet comandi

Si lanciano con .\bot.ps1 <comando> · bot.ps1 ricompila da solo quando serve.

# Come l'hai fatto girare (config attuale: no-proxy, canary off, fuori orario)
.\bot.ps1 unquarantine
$env:BYPASS_WORKING_HOURS='true'; $env:SELECTOR_CANARY_ENABLED='false'; .\bot.ps1 sync-list --no-proxy

🔑 Setup / accesso

.\bot.ps1 loginLogin manuale LinkedIn (apre il browser)
.\bot.ps1 doctorDiagnostica: proxy, cookie, login, salute account
.\bot.ps1 test-connection [--no-proxy]Testa latenza proxy + raggiungibilità LinkedIn
.\bot.ps1 proxy-statusStato del proxy (config + failover chain)

⚙️ I workflow azioni su LinkedIn · anti-ban

.\bot.ps1 sync-listImporta una lista SalesNav → DB + email + score
.\bot.ps1 sync-searchEstrae dalle ricerche salvate
.\bot.ps1 send-invitesInvia richieste di collegamento (nota AI)
.\bot.ps1 send-messagesFollow-up a chi ha accettato
.\bot.ps1 autopilotPilota automatico (cicla tutto) · alias run-loop
.\bot.ps1 random-activity --max-actions 3Azioni leggere (scroll feed), zero inviti

Flag utili: --no-proxy opera dal tuo IP reale (no proxy) · $env:BYPASS_WORKING_HOURS='true' opera fuori orario · $env:SELECTOR_CANARY_ENABLED='false' salta il canary pre-sessione.
Quality gate sviluppo: npm run conta-problemi (typecheck + lint + test).
Anti-ban: i comandi azioni su LinkedIn aprono il browser e agiscono sull'account — usali con criterio (orari plausibili, proxy coerente, volumi sotto i cap).
