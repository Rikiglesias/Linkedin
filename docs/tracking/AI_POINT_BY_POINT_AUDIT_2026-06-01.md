# Audit Zero-Trust del Cervello AI — 2026-06-01

## Metodo zero-trust

Un punto vale chiuso solo se ha regola esplicita, meccanismo applicato, audit/verifica e fonte aggiornata. Checkbox, backlog e memoria non sono prove da soli. Se manca uno dei quattro elementi, lo stato resta `parziale`, `aperto reale`, `obsoleto/duplicato` o `da rendere piu esplicito`.

Benchmark usati: Claude Code memory e hooks, OpenAI Codex agent loop, How OpenAI uses Codex, MCP security best practices, Obsidian Bases syntax. Sintesi applicata: memoria/CLAUDE/AGENTS sono contesto, non enforcement; hook e audit sono enforcement; Codex usa AGENTS/skill/env nel loop; MCP va least-privilege; `.base` deve restare YAML valido.
