## Note operative (manuali, preservate alla rigenerazione)

- `AI_PROVIDER` (F0 ai-stack): `auto` | `anthropic` | `openai` | `ollama` | `template`; default `auto` = chain storica. Guard zero-PII attiva: i purpose con dati lead restano su endpoint locale anche con provider cloud esplicito.
- `ANTHROPIC_API_KEY`: richiesta se `AI_PROVIDER=anthropic`, insieme a `AI_ALLOW_REMOTE_ENDPOINT=true`.
- `ANTHROPIC_MODEL` (default `claude-opus-4-8`): tier **brain** — cervello/decisioni (decision_engine, guardian, ai_advisor).
- `ANTHROPIC_MODEL_LIGHT` (default `claude-haiku-4-5-20251001` — F2 ai-stack): tier **light** — generazione no-PII a basso valore decisionale (decoy_terms, post_content), ~1/5 del costo del brain.
- `ANTHROPIC_TIMEOUT_MS` (default 60000): applicato dal costruttore SDK.
- `COMPUTER_USE_MODEL` (default `gpt-5.4` — F2): model id computer-use centralizzato (era costante hardcoded in computerUse.ts).
- `VISION_ALLOW_CLOUD` (default `false` — F2 zero-PII): opt-in esplicito all'uscita di SCREENSHOT verso provider cloud (vision OpenAI + computer-use); richiede anche `AI_ALLOW_REMOTE_ENDPOINT=true`. Di default gli screenshot LinkedIn non lasciano la macchina (decisione 2026-06-11; attivare solo con DPA/zero-retention/informativa a posto).

### Formato File Proxy List

Il file `PROXY_LIST` supporta piu' formati. Uno per riga:

```
# Formato URL standard
http://user:pass@proxy1.example.com:8080

# Formato host:port:user:pass
proxy3.example.com:8080:user:pass

# Con prefisso tipo (raccomandata)
residential|http://user:pass@proxy1.example.com:8080
mobile|http://user:pass@proxy2.example.com:8080

# Le righe con # sono ignorate
# I duplicati vengono rimossi automaticamente
```
